## 基本概念

此 SKILL 會自動識別檔案中的個人資訊欄位，並填入以下資料：

```json
{
  "name": "曾慶良",
  "phone": "0981737608",
  "address": "新北市中和區崇南里14鄰忠孝街126-3號1樓",
  "id_number": "F123614400",
  "service_unit": "新興科技推廣中心(永春高中)",
  "position": "主任"
}
```

## 各檔案類型的填寫方法

### Word 文件 (.docx)

**方法 1：使用預留位置**
在文件中插入預留位置，支援以下格式：
- `[姓名]` → 曾慶良
- `[電話]` → 0981737608
- `{address}` → 新北市中和區崇南里14鄰忠孝街126-3號1樓
- `{{position}}` → 主任

**方法 2：使用表單欄位**
在 Word 中插入表單欄位，我會自動填寫相符的欄位。

**方法 3：使用表格**
建立表格並將欄位名稱作為表頭，我會自動填寫相應列。

### Excel 試算表 (.xlsx)

**方法 1：指定儲存格**
告訴我儲存格位置，例如：
- A1: 姓名
- A2: 電話
- A3: 地址

**方法 2：使用表頭匹配**
在 Excel 中設置表頭（如「姓名」、「電話」、「地址」），我會自動填寫下方的儲存格。

**範例：**
```
| 姓名 | 電話 | 地址 | 身分證號 | 服務單位 | 職稱 |
|-----|-----|------|--------|--------|------|
| [待填] | [待填] | [待填] | [待填] | [待填] | [待填] |
```

### PDF 表單 (.pdf)

1. 確認 PDF 包含表單欄位（不是掃描的圖片）
2. 我會識別所有表單欄位並填寫相符的欄位
3. 生成已填寫的 PDF 副本

### HTML 表單

**方法 1：具有 name 屬性的輸入欄位**
```html
<input type="text" name="姓名" />
<input type="tel" name="電話" />
<input type="text" name="地址" />
```

**方法 2：具有 label 的輸入欄位**
```html
<label>姓名：<input type="text" /></label>
<label>電話：<input type="tel" /></label>
```

### JSON 檔案

直接在 JSON 中填寫相應欄位：

```json
{
  "user": {
    "name": "",
    "phone": "",
    "address": "",
    "id_number": "",
    "service_unit": "",
    "position": ""
  }
}
```

### 純文字檔案

使用以下任一格式的預留位置：
- `{姓名}`
- `[電話]`
- `__地址__`
- `${身分證號}`
- `{{服務單位}}`

## 進階用法

### 1. 條件填寫
指定要填寫的欄位，例如：
- 「只填寫姓名和電話」
- 「填寫除了身分證號之外的所有欄位」

### 2. 自訂格式
提供特定的格式需求，例如：
- 「地址前加『住址：』」
- 「電話使用格式 (0981) 737-608」

### 3. 批量填寫
同時提供多個檔案，我會逐個填寫並返回結果。

### 4. 驗證和確認
在最終保存前，我會顯示即將填寫的內容以供確認。

## 故障排除

| 問題 | 解決方案 |
|-----|--------|
| 「找不到欄位」| 確認欄位名稱或提供更多資訊 |
| 「格式不受支援」| 提供其他格式的檔案或告知欄位位置 |
| 「填寫不完整」| 確認所有需填寫的欄位名稱 |
| 「檔案受保護」| 提供未受保護版本或告知密碼 |

## 常見使用場景

### 場景 1：簽署表格
```
使用者：我有一份需要簽署的表格，請幫我填寫個人資訊。
我：我會填寫姓名、地址、身分證號等相關欄位。
```

### 場景 2：報名表單
```
使用者：請幫我填寫這份報名表。
我：我會自動填寫所有個人資訊欄位。
```

### 場景 3：批量處理
```
使用者：我有 5 份申請表需要填寫個人資訊。
我：我會逐個填寫每份表格並提供所有已填完成的檔案。
```

## 輸出說明

完成填寫後，我會：

1. **顯示摘要** - 確認填寫的欄位和內容
2. **提供檔案** - 返回已填寫的檔案
3. **驗證報告** - 指出任何無法填寫的欄位（如有）

## 隱私與安全

- 個人資料只用於填寫你提供的檔案
- 不會上傳或分享你的資料
- 原始檔案保持不變，只提供已填寫的副本

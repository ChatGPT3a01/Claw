# 客戶需求分析器

## 執行步驟
### Step 1：需求識別
- 識別所有明確提出的需求項目
- 識別隱含的（未直接說明但可推斷的）需求

### Step 2：需求分類
- 核心需求：客戶明確要求，必須滿足
- 期望需求：客戶希望有，但非必要
- 潛在需求：AI 建議的加分項目

### Step 3：風險標記
- Check if 需求涉及第三方整合 → 標記「需協調」
- Check if 時程低於 2 週 → 標記「時程風險」

### Step 4：產出結構化摘要（此格式供 proposal-drafter 使用）
- MUST 包含核心需求表格（含優先級）
- NEVER 遺漏客戶明確提到的任何需求

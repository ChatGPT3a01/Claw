💾 11.1 匯出為圖片檔案

fig.savefig() 是最基本的儲存方式。

import matplotlib.pyplot as plt

fig, ax = plt.subplots()
ax.plot([1, 2, 3], [1, 4, 9])

# 儲存為 PNG (預設)
fig.savefig("my_plot.png")

# 儲存為 JPG
fig.savefig("my_plot.jpg")

# 儲存為 PDF (向量圖)
fig.savefig("my_plot.pdf")
向量圖優勢： PDF 或 SVG 格式可以無限放大不失真，適合用於印刷或大螢幕投影。

✨ 提升圖片品質：DPI 與 Bbox

預設的圖片往往解析度不足，且周圍有太多留白。

# dpi=300: 設定高解析度 (Dots Per Inch)
# bbox_inches='tight': 自動裁切多餘白邊
fig.savefig("high_quality.png", dpi=300, bbox_inches="tight")
參數 說明 建議值
dpi 解析度 300 (印刷), 72 (網頁)
bbox_inches 邊界裁切 "tight" (強烈推薦)

👻 透明背景匯出

如果你要將圖表貼到深色背景的簡報中，透明背景非常重要。

# transparent=True
fig.savefig("transparent_plot.png", transparent=True, dpi=300)
注意：JPG 不支援透明背景，請使用 PNG 或 PDF。

💻 隨堂測驗 1：存檔參數

任務： 請補全程式碼，將圖片儲存為 300 DPI 的 PNG，並裁切掉多餘白邊。

fig.savefig("chart.png", dpi=_____, bbox_inches="_______")

請填入正確的數值與字串。

✅ 測驗 1 解答

fig.savefig("chart.png", dpi=300, bbox_inches="tight")
這兩個參數是存檔時的「標配」，建議養成習慣加上去。

📑 11.2 匯出為 PDF 報表

有時候我們需要將多張圖表整合到一份文件中，而不是產生一堆零散的 PNG。

matplotlib.backends.backend_pdf.PdfPages 可以達成這個需求。

from matplotlib.backends.backend_pdf import PdfPages

# 開啟一個 PDF 檔案準備寫入
with PdfPages('report.pdf') as pdf:
    # ... 繪圖邏輯 ...
    # pdf.savefig(fig)

📄 實作：多頁 PDF 報告

import numpy as np
from matplotlib.backends.backend_pdf import PdfPages

with PdfPages('multipage_report.pdf') as pdf:
    # --- 第 1 頁 ---
    fig1, ax1 = plt.subplots(figsize=(8, 6))
    ax1.plot(np.random.randn(50))
    ax1.set_title("Page 1: Trend")
    pdf.savefig(fig1)  # 儲存當前 Figure 到 PDF
    plt.close(fig1)    # 關閉 Figure 釋放記憶體

    # --- 第 2 頁 ---
    fig2, ax2 = plt.subplots(figsize=(8, 6))
    ax2.hist(np.random.randn(100))
    ax2.set_title("Page 2: Distribution")
    pdf.savefig(fig2)
    plt.close(fig2)

🏷️ 加入 PDF 元數據 (Metadata)

讓報告更專業，我們可以設定標題、作者等資訊。

with PdfPages('report.pdf') as pdf:
    # ... 繪圖 ...
    
    # 設定元數據
    d = pdf.infodict()
    d['Title'] = 'Monthly Sales Analysis'
    d['Author'] = 'Data Team'
    d['Subject'] = 'Q3 Performance'
    d['CreationDate'] = datetime.datetime.today()

🧠 隨堂測驗 2：PDF 製作流程

任務： 請排列製作多頁 PDF 的正確步驟：

  1. pdf.savefig(fig)
  2. with PdfPages('file.pdf') as pdf:
  3. fig, ax = plt.subplots()
  4. plt.close(fig)

正確順序是？

  1. 2 -> 3 -> 1 -> 4
  2. 3 -> 1 -> 4 -> 2
  3. 2 -> 1 -> 3 -> 4
  4. 1 -> 2 -> 3 -> 4

✅ 測驗 2 解答

答案：(1) 2 -> 3 -> 1 -> 4

流程：

  1. 開啟 PDF 檔案環境。
  2. 建立圖表 (Figure)。
  3. 將圖表存入 PDF。
  4. 關閉圖表 (釋放記憶體)。

📓 11.3 在 Jupyter/Colab 中呈現

Jupyter Notebook 本身就是一份很好的報告，只要善用 Markdown。

  • 使用 # 建立清晰的標題層級。
  • 使用文字描述分析洞察，而不僅僅是貼程式碼。
  • 適時隱藏冗長的程式碼區塊。

🌐 匯出為 HTML 網頁

這是分享給「沒有安裝 Python 環境」的人最好的方式。

Jupyter Notebook

File > Download as > HTML (.html)

Google Colab

Colab 沒有直接匯出 HTML 的按鈕，需透過指令：

!jupyter nbconvert --to html your_notebook.ipynb

🙈 隱藏程式碼 (Clean Report)

給老闆看的報告，通常不需要顯示複雜的 Python 程式碼。

在匯出時，可以使用 --no-input 參數。

# 在終端機或 Colab 單元格執行
!jupyter nbconvert --to html --no-input analysis.ipynb
這樣生成的 HTML 報告將只包含Markdown 文字和圖表結果，非常整潔。

🧠 隨堂測驗 3：報告格式

情境： 你完成了一份分析，需要寄給行銷部門的同事看。他們不懂 Python，也不想安裝任何軟體，只想在瀏覽器上看到圖表和結論。

任務： 你應該匯出成什麼格式？

  1. .ipynb (Notebook 檔)
  2. .py (Python 腳本)
  3. .html (網頁檔)
  4. .txt (純文字)

✅ 測驗 3 解答

答案：(3) .html (網頁檔)

解釋： HTML 檔案包含了所有的圖表和文字，可以用任何瀏覽器 (Chrome, Edge) 開啟，完全不需要 Python 環境，是分享靜態報告的最佳選擇。

☁️ 11.4 Google Colab 分享

如果是團隊協作，或者希望對方能執行程式碼：

  1. 點擊右上角的 [共用] (Share) 按鈕。
  2. 設定權限：
    • 檢視者 (Viewer)：只能看，不能改。
    • 評論者 (Commenter)：可以留註解。
    • 編輯者 (Editor)：可以修改程式碼。
  3. 複製連結寄給對方。

🐱 GitHub 分享

將 .ipynb 檔案上傳到 GitHub，GitHub 會自動渲染 Notebook。

注意： GitHub 有時渲染會比較慢或失敗。如果 GitHub 失敗，可以將 GitHub 的連結貼到 nbviewer.jupyter.org 來查看。

💻 隨堂測驗 4：協作模式

問題： 在 Google Colab 中，如果你希望對方能看到你的程式碼並執行，但不希望對方不小心改壞你的原始檔案，你應該設定什麼權限？

  1. 編輯者 (Editor)
  2. 檢視者 (Viewer)
  3. 擁有者 (Owner)
  4. 封鎖者 (Blocker)

✅ 測驗 4 解答

答案：(2) 檢視者 (Viewer)

解釋： 設定為「檢視者」後，對方可以打開並執行程式碼（在他們自己的暫存環境中），但無法將修改存回你的原始檔案，這是最安全的分享方式。

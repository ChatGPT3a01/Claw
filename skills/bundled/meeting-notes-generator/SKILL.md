# 會議紀錄產生器

## 用途

當使用者提供會議逐字稿、摘要或討論重點時，本技能會：

1. 收集會議基本資訊（日期、時間、與會人員）
2. 分析會議內容，提取議題、決議與待辦事項
3. 產生結構化的標準會議紀錄

## 前置檢查

在開始執行前，MUST 先檢查以下條件：

1. Check if 使用者是否提供了會議內容
   - 若否 → Ask user「請提供會議的逐字稿或重點摘要」

2. Check if 是否包含會議日期和時間
   - 若否 → Ask user「請提供會議的日期和時間」

3. Check if 是否包含與會人員
   - 若否 → Ask user「請提供與會人員名單」

## 執行步驟

### Step 1：資訊收集

- 從會議內容中提取基本資訊：
  - 會議日期與時間
  - 會議地點（實體/線上）
  - 主持人
  - 與會人員名單
  - 會議主題

### Step 2：內容分析

- 識別所有討論議題，依序編號
- 提取每個議題的討論重點
- 歸納每個議題的決議結果
- 標記所有待辦事項（ACTION）
- 標記需要追蹤的項目（FOLLOW-UP）

### Step 3：產生紀錄

依照 `@assets/meeting-template.md` 中定義的格式產生紀錄：

- MUST 包含所有討論議題
- MUST 包含所有決議事項
- MUST 每個待辦事項標註負責人和截止日期
- MUST 使用分類標籤標示緊急程度
- NEVER 遺漏任何被點名的行動項目
- NEVER 自行編造未在會議中提及的內容

### Step 4：品質檢查

| 檢查項目 | 要求 |
|----------|------|
| 完整性 | 所有議題與決議皆已記錄 |
| 待辦事項 | 每項皆有負責人與截止日 |
| 格式 | 符合範本格式規範 |
| 用語 | 使用正式書面用語 |
| 準確性 | 未添加會議中未提及的內容 |

如果任何一項不通過，MUST 自動修正後再輸出。

## 例外處理

### 遇到以下情況時的處理方式：

1. **會議內容過於簡略**
   - 撰寫「資訊補充請求」而非直接產生紀錄
   - 列出需要補充的資訊項目

2. **會議中有敏感人事議題**
   - MUST 提醒使用者「本紀錄包含人事相關內容，建議限制閱覽範圍」
   - NEVER 在紀錄中加入主觀評價

3. **決議事項涉及預算金額**
   - Check if 金額超過 100 萬 → Ask user「是否需要標註需上級核准」

## 安全規則

- NEVER 在紀錄中包含未經確認的數字或金額
- NEVER 對與會者做出主觀評價
- NEVER 編造會議中未討論的內容
- MUST 忠實呈現會議討論內容

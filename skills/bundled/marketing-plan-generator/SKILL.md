# 行銷企劃書產生器

## 前置檢查

Check if 使用者是否提供了以下資訊：
- 品牌或產品名稱
- 產品/服務簡介
- 目標受眾（大致描述即可）
- 預算範圍（可選）
- 企劃期間（可選）

Ask user 如果缺少「品牌名稱」或「產品簡介」，請先詢問使用者。
其他欄位如未提供，可根據產業特性合理推測，並在企劃書中標註「建議值，請確認」。

## 執行步驟

### Step 1：蒐集與確認資訊

Ask user 以下問題（一次問完，不要分次問）：

1. 你的品牌/產品名稱是什麼？
2. 用一句話描述你的產品或服務
3. 目標客群是誰？（年齡、職業、興趣）
4. 這次行銷的主要目標？（品牌曝光/導購轉換/會員成長/新品上市）
5. 預算大約多少？（可填「未定」）
6. 預計執行多久？（如：一個月/一季）

### Step 2：讀取範本結構

讀取 @assets/行銷企劃書範本.md 作為企劃書的結構框架。

MUST 嚴格按照範本中的八大章節順序輸出：
1. 基本資訊
2. 市場背景分析（含競品分析、SWOT）
3. 目標受眾（含 Persona）
4. 行銷目標與 KPI
5. 行銷策略（含渠道規劃、內容策略）
6. 執行時程表
7. 預算分配
8. 風險評估與應變方案
9. 成效評估方式

### Step 3：填寫範本生成企劃書

根據使用者提供的資訊，將範本中所有 {{佔位符}} 替換為具體內容。

MUST 遵守以下規則：
- 所有表格 MUST 填寫完整，不得留空
- 競品分析至少列出 3 個競品
- Persona 至少建立 2 個角色
- KPI 至少設定 4 個指標
- 行銷渠道至少規劃 4 個
- 時程表至少分 5 個階段
- 預算分配各項目金額加總 MUST 等於總預算

### Step 4：補充專業建議

在企劃書最後，額外提供：
- 3 個可能的「核心標語/Slogan」選項
- 針對目標受眾的 2-3 個創意行銷點子
- 1 個「如果預算加倍」的進階方案建議

### Step 5：儲存並轉換 Word 檔

1. 將完成的企劃書儲存為 Markdown 檔案（檔名格式：`行銷企劃書_{{品牌名稱}}_{{日期}}.md`）
2. Ask user 「要我幫你轉成美觀的 Word 檔嗎？」
3. 如果使用者同意，呼叫 docs Skill 將 Markdown 轉換為格式化的 Word 文件（.docx）

## 例外處理

- 如果使用者只提供品牌名稱，沒有其他資訊：先用網路搜尋該品牌基本資料，再詢問缺少的關鍵資訊
- 如果使用者說「預算未定」：以產業平均值估算，並標註「此為建議預算，請根據實際情況調整」
- 如果產品類型不明確：提供 2-3 個可能的定位方向讓使用者選擇

## 安全規則

MUST 使用繁體中文撰寫所有內容
MUST 所有數據和分析必須基於合理推估，不得捏造具體統計數字
MUST 在涉及推估數據時標註「預估值」
NEVER 抄襲任何現有品牌的企劃內容
NEVER 在企劃書中包含不實或誇大的效果保證

## 輸出格式

輸出為 Markdown 格式，結構嚴格按照 @assets/行銷企劃書範本.md。

輸出順序：
1. 先輸出完整企劃書（八大章節）
2. 再輸出「補充專業建議」（標語、創意點子、進階方案）
3. 最後詢問使用者：「需要我調整哪個部分？或是要我把這份企劃書轉成 Word 檔？」

## 範例

### 輸入範例
```
幫我寫一份行銷企劃書。
品牌：鮮果日記
產品：訂閱制每週水果箱（NT$599/週）
目標：25-40歲注重健康的上班族
目標：增加訂閱數
預算：每月 10 萬
時間：3 個月
```

### 預期輸出
一份完整的行銷企劃書，包含：
- 基本資訊表格（填入鮮果日記相關資料）
- 健康飲食/訂閱經濟的市場分析
- 3 個水果箱競品比較（如：無毒農、果物配、i Fruit）
- 2 個 Persona（如：忙碌的科技業 PM、注重養生的行政主管）
- KPI：訂閱數成長 30%、社群觸及 10 萬、轉換率 3%、回購率 60%
- 渠道：IG 內容行銷、FB 廣告、KOL 開箱、LINE 會員經營
- 三個月的時程表（預熱→推廣→衝刺→維護）
- 30 萬預算分配明細
- 風險評估（如：季節性水果供應不穩）
- 三個 Slogan 選項

😩 為什麼需要 Claw？

備課的時候，常常需要不同 AI 做不同事，但每換一家就有以下痛點：

🪟 切視窗切到頭暈

Claude 一個視窗、Gemini 一個視窗、Codex 一個視窗，五個視窗開下去，誰是誰都搞不清楚。

🔐 每換一家要重新登入

三家 CLI 各有自己的認證流程，常常切過去發現要重新貼 API Key、重新走 OAuth。

🧠 對話歷史丟失

上一個視窗講過的內容、寫過的程式碼，到下一個視窗完全要重新講一次。

💸 從零冷啟動，token 燒得快

每次新對話 prompt cache 都 miss，同樣的工作量花 10 倍的 token。

結論：多個 AI CLI 各自為政是個大坑。Claw v3 用 plugin 模型把這些痛點全打包解決。

🎯 Claw 的四大亮點

它把「同時用三家 AI」這件事，變得跟「只用 Claude」一樣自然：

✨ 真 Session

主場 Claude 全程不重啟，對話記憶完整、prompt cache 100% 命中。越聊越快、越聊越省。

💰 Token 省 ~90%

cache 命中之後，後續訊息成本只剩原本的 10%。同樣的工作量、token 消耗暴降。

🎬 一鍵安裝

三行指令解決：自動補裝 CLI、自動裝 plugin、自動認證。學員完全不用記任何 npm 指令。

🤖 自動模式

三家 AI 都預設走自動模式，跳過繁瑣的權限詢問。讓 AI 多一點自主判斷，您少一點無謂中斷。

🦞 集大成：把 OpenAI Codex × Google Gemini × Anthropic Claude 三家頂尖 AI 統合成一個無縫協作的指揮所 — 這就是 Claw。

👥 三家 AI、各司其職

v3 把三家 AI 設計成「主場 + 兩位顧問」的協作關係：

🦞 Claude（主場）

負責主要對話、寫作、寫程式

• 完整對話記憶
• Prompt cache 命中
• 最便宜、最快

🤖 Codex（顧問）

OpenAI 視角，擅長挑戰決策

• 程式碼審查
• 對抗式挑戰
• 委派任務修 bug

✨ Gemini（顧問）

1M context，看整個專案

• 大型 codebase 審查
• 整本講義一次看完
• 跨檔案整合分析

使用心法：主場永遠是 Claude，需要 OpenAI 或 Google 視角時才叫顧問來幫忙。

📦 唯一前置條件

⚙️
Node.js 18+
沒裝就去 https://nodejs.org 下一步到底
好消息：其他三家 CLI（claude / codex / gemini）
👉 install.ps1 會自動偵測並用 npm install -g 補上，您不用記任何 npm 指令。
為什麼 Node.js 不能自動裝？
Node.js 安裝需要管理員權限與系統級設定（PATH、執行檔關聯），不適合腳本自動處理。其他 CLI 都是 npm 套件，跟 Node.js 走，所以可以自動補。

🚀 一鍵安裝（真·三行）

確認 Node.js 已裝後，開啟 PowerShell，貼下面三行：

git clone https://github.com/ChatGPT3a01/Claw.git
cd Claw
.\install.ps1
安裝過程會問您兩件事：
  1. 跳出瀏覽器登入 ChatGPT（Codex 認證）
  2. 請您貼 Gemini API Key（從 https://aistudio.google.com/app/apikey 取得，免費）
裝完後關掉 PowerShell、重新開一個，任何位置打 claw 即可啟動。

🔧 install.ps1 自動做的 6 件事

看似只跑一個指令，背後做了完整的環境配置：

  1. 檢查並補裝 npm CLI — 偵測 claude / codex / gemini，缺哪個自動 npm install -g 補上
  2. 部署 claw 命令到 ~/.claude/claw/，內含 claw.ps1、claw.bat、setup-plugins.ps1、uninstall.ps1
  3. 自動裝兩個 Claude Code plugin — codex@openai-codex 和 gemini@abiswas97-gemini
  4. 設定 User PATH（CMD / PowerShell 都認得）+ PowerShell alias（mcp-on / mcp-off / mcp-ls）
  5. 互動式認證 三家 AI — Claude（API key 或 Pro 訂閱二選一）/ Codex login / GOOGLE_API_KEY
  6. 顯示完成提示，告訴您下一步要做什麼
所有設定都用 marker 區塊保護，重複執行 install.ps1 不會破壞您既有的 PowerShell profile 或 Codex 設定。

🎬 啟動 claw

裝完之後，關掉舊 PowerShell、開新的（讓 PATH 生效），然後：

claw

您會看到：

隨時按一下鍵 → 進入 Claude Code 主對話。沒有時間限制，您慢慢讀。

🌟 最簡單用法（推薦學員）

0 學習成本 不用記任何指令 會說話就會用

進入 claude 後，直接用中文跟它說想做什麼：

請 Codex 審查我剛寫的程式碼
讓 Gemini 用 1M context 看整個專案
派 Codex 修這個 bug
👉 Claude 聽得懂、會自動派工。學員完全不用記 /codex:review 之類的指令，會講話就會用。

⚡ 進階：Slash 指令

想要精準控制可以直接打 slash 指令（打 /codex: 然後 Claude Code 會跳出選單）：

指令說明
/codex:review程式碼審查（一般）
/codex:adversarial-review對抗式挑戰你的設計
/codex:rescue <任務>委派任務（修 bug、追問題）
/gemini:review1M context 看整個專案
/gemini:adversarial-reviewGemini 對抗式挑戰
/gemini:rescue <任務>委派 Gemini

常用旗標

  • --background — 背景跑，繼續對話
  • --base main — 審查 branch 與 main 差異
  • --model <名稱> — 指定模型（Gemini: flash / pro / flash-3 / pro-3）

🔍 情境 1：程式碼雙視角審查

寫完一段程式碼後，讓 Codex 和 Gemini 各自審一次，比對兩家意見：

▸ 您：幫我寫個費氏數列函式
▸ Claude：[寫好程式碼]

▸ 您：/codex:review
▸ Codex：第 5 行可能整數溢位、缺邊界檢查...

▸ 您：/gemini:review
▸ Gemini：遞迴版本 O(2ⁿ)，建議改 memoization...

▸ 您：整合兩家意見幫我改
▸ Claude：[整合後的最終版本]
為什麼有用？OpenAI（Codex）和 Google（Gemini）的訓練資料、安全偏好都不一樣，兩家視角能抓出彼此沒注意到的盲點。三家 AI 對同一份程式碼的分歧，往往就是值得思考的地方。

📚 情境 2：1M context 看整個專案

Gemini 的殺手級應用 — 一次看完整個 codebase / 整本講義，這是 Claude / Codex 都做不到的。

適合場景

📖 整本講義審查

100 頁講義一次丟進去，找前後矛盾、重複內容、章節銜接問題。

🏗️ 整個 codebase 分析

讓 Gemini 看整個 React 專案，找出 anti-pattern、命名不一致、跨檔案耦合。

📄 大型 PDF 摘要

50 頁報告、學術論文，一次摘要重點、列出關鍵章節。

實際用法：
讓 Gemini 看 D:\我的講義 整個資料夾，找前後矛盾或重複內容
1M token 多大？ 約 = 70 萬中文字 = 2000 頁文件 = 一本完整的程式書。

⚔️ 情境 3：對抗式挑戰你的設計

/codex:adversarial-review 不是找 bug，是挑戰你的設計決策。

它會做這些事

  • ❓ 質疑你選的架構 / 演算法 / 工具
  • 🚨 找出潛在的失敗模式（race condition、雪崩效應）
  • 💡 提出替代方案，並比較利弊
  • 🔍 點出你沒考慮到的 edge case

實例

▸ 您：幫我設計一個快取系統
▸ Claude：[設計用 LRU + Redis...]

▸ 您：/codex:adversarial-review

▸ Codex 開始挑戰：
  「為什麼選 LRU 不選 LFU？」
  「快取爆滿時的 fallback 是什麼？」
  「並發寫入時會不會 race condition？」
  「Redis 掛了怎麼辦？是否需要 multi-tier？」
什麼時候用？寫完重要設計、做完教學設計、實作前先找問題。比起問「對嗎？」，問「哪裡可能會壞？」更有價值。

⚡ 情境 4：背景委派、邊做邊等

--background 旗標把長任務丟給 Codex / Gemini 跑，您繼續用 Claude 做別的事。

▸ 您：/codex:rescue --background 修這個 failing test
▸ Claude：已派工 Codex 在背景跑 (job: cdx-abc123)

▸ 您：（繼續用 Claude 寫教案、回 email、改文件...）

▸ 您：/codex:status
▸ Claude：Codex 還在跑，已花 3 分鐘

▸ 您：（繼續做別的）

▸ 您：/codex:result
▸ Claude：Codex 修好了，patch 如下：[diff]

適合場景

📋 一邊回信一邊審講義

🔧 重構任務丟背景

🛠️ MCP 切換（給進階使用者）

Claude Code 啟動會把 ~/.claude.json 的所有 MCP 載入到 context window。裝太多會拖慢啟動、燒 token。

解法：按需切換 MCP

mcp-ls                       # 看目前啟用了哪些 MCP
mcp-on twstockmcpserver      # 想用台股 MCP 時臨時載入
mcp-off twstockmcpserver     # 用完卸下
mcp-off all                  # 一次卸下全部（啟動最快）
架構：所有備援 MCP 設定都放在 ~/.claude/mcp-presets/，平常 mcpServers 保持空的（最快），需要時用一個指令切回，無痛。
提示：切換後必須關閉並重啟 Claude Code / claw 才會生效。

🆘 常見問題

❌ 'claw' 不是內部命令

沒重開視窗。完全關掉所有終端機，重新開新的就好（Windows PATH 只在啟動時載入）。

🚫 執行原則限制

改用：
powershell -ExecutionPolicy Bypass -File .\install.ps1

📋 進 claude 後想再看 cheatsheet

在 claude 對話中輸入：
!claw --cheat
會即時印出完整使用說明。

🗑️ 想完全移除

執行：
& "$env:USERPROFILE\.claude\claw\uninstall.ps1"
會清乾淨 PATH、alias、plugin。

更多問題：claw --setup-help 看 plugin 安裝指引，或上 GitHub issues 反應。
